FOC 02 — Clarke Transform (abc → αβ)

Goal: turn three-phase values (a,b,c) into a 2D vector in a stationary plane (α,β). This makes the system easier to visualize and prepares us for Park (αβ → dq).

On this page 1) Why do we need Clarke? 2) The idea (high level) 3) Interactive widget 4) The standard equations (reference) 5) Next

1) Why do we need Clarke?

In a balanced 3-phase system, the three signals are not independent: knowing two is enough (the third is implied). Clarke gives us a clean 2D representation: a single vector in the αβ plane.

After Clarke, we stop thinking “three separate phase values” and start thinking: one rotating vector in a stationary plane.

2) The idea (high level)

Clarke is basically a coordinate change: we project the 3-phase system onto two perpendicular axes (α and β).

Later, Park will rotate this αβ plane into the rotating dq frame.

Interactive: Clarke (abc → αβ)

Move phase a and phase b. We assume balance, so c = −a − b. Watch the resulting αβ vector.

abc sliders → αβ vector

This is the Clarke transform in action (simple, visual).

3) The standard equations (reference)

For now, just keep these as a reference. We’ll build intuition and visuals next.

Assuming a balanced system (a+b+c=0), one common form is:

α = a
β = (1/√3) · (a + 2b)

There are other equivalent forms depending on scaling (power-invariant, amplitude-invariant, etc.). We’ll choose one convention and stick to it consistently across all pages.
